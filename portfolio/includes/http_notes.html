What is HTTP?

HTTP is a syntactic protocol that is fundamental to the current interoperability of the web, providing a standardised mechanism for meaningful client-server communication, and consequently the sharing of resources. The client sends requests to the server, which sends responses back and potentially resources such as HTML documents, images and APIs eventually follow (the transport of these resources not being related to HTTP directly, but indirectly). This communication is done via individual messages meaning the process is stateless, with HTTP cookies allowing for stateful sessions. The Internet Protocol Model and the OSI model define HTTP as sitting in the application layer, a layer specifying shared communication protocols.

How Does HTTP Work?

Establishing a Connection

The client initiates a connection to the server. This involves initiating a connection in the underlying transport layer, often done using TCP ( a Transmission Control Protocol).

Sending a Client Request

With a connection in place, the user-agent can send the request. For example: 

POST /contact_form.php HTTP/1.1
Host: developer.mozilla.org
Content-Length: 64
Content-Type: application/x-www-form-urlencoded

name=Joe%20User&request=Send%20me%20one%20of%20your%20catalogue

GET and POST methods are the most common request methods - used for retrieving data and requesting changes to its state respectively.

Server Response

Server side, the request is processed and ultimately a response is sent back. The response also consists of text directives, with a status line acknowledging the HTTP version used and further information for the client regarding the data sent (e.g. compression algorithm). It may also include a data block - the below is an example of a successful web page response: 

HTTP/1.1 200 OK
Content-Type: text/html; charset=utf-8
Content-Length: 55743
Connection: keep-alive
Cache-Control: s-maxage=300, public, max-age=0
Content-Language: en-US
Date: Thu, 06 Dec 2018 17:37:18 GMT
ETag: "2e77ad1dc6ab0b53a2996dfd4653c1c3"
Server: meinheld/0.6.1
Strict-Transport-Security: max-age=63072000
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
Vary: Accept-Encoding,Cookie
Age: 7

Why is HTTP important?

HTTP defines exact messages and meanings (methods, headers, status codes, message bodies) involved in the communication between client and server interactions. The standardisation of this system is what makes it so important as it enables communication across the web without two individual points having to define their own communication protocol. Whilst other communication protocols exist in the application layer, the ubiquity of HTTP, and its extensibility, enable communication standardisation across the web.

Common HTTP Methods

  • GET: The GET method requests a representation of the specified resource. Requests using GET should only retrieve data and should not contain a request content.
  • HEAD: The HEAD method asks for a response identical to a GET request, but without a response body.
  • POST: The POST method submits an entity to the specified resource, often causing a change in state or side effects on the server.
  • PUT: The PUT method replaces all current representations of the target resource with the request content.
  • DELETE: The DELETE method deletes the specified resource.
  • CONNECT: The CONNECT method establishes a tunnel to the server identified by the target resource.
  • OPTIONS: The OPTIONS method describes the communication options for the target resource.
  • TRACE: The TRACE method performs a message loop-back test along the path to the target resource.
  • PATCH: The PATCH method applies partial modifications to a resource.
